## 1. Цель MVP

Показать трекеру **рабочую систему end-to-end**:

1. **Пайплайн** — загрузка прайса → парсинг → сопоставление со справочником → запись в БД  
2. **Поиск** — услуга → список клиник с ценами (резидент / нерезидент)  
3. **Админка** — статусы документов, unmatched-очередь, метрики качества  

Демо должно работать **на реальных данных** из папки `прайсы/` и `Справочник услуг.xlsx`.

## 4. План по фазам

### Фаза 0 — Подготовка (0–2 ч) ✅ частично готово

- [x] Виртуальное окружение `.venv`, библиотеки парсинга
- [ ] Структура репозитория (monorepo: `backend/`, `frontend/`, `docs/`)
- [ ] `docker-compose.yml`: API + PostgreSQL + (опционально) frontend
- [ ] `partners.json` — статический конфиг 8 клиник (город, адрес, телефон, email)
- [ ] `.env.example`: `DATABASE_URL`, `GEMINI_API_KEY`

### Фаза 1 — Ядро бэкенда (2–4 ч)

**Приоритет: без этого ничего не работает**

| Задача | Результат |
|---|---|
| SQLAlchemy-модели | Partner, Service, PriceDocument, PriceItem |
| Alembic / init migrations | Схема БД по ТЗ |
| Импорт справочника | 1286 услуг в таблицу `services` |
| Импорт partners.json | 8 партнёров в таблицу `partners` |
| Базовые эндпоинты | `GET /services`, `GET /partners`, `GET /health` |

**Критерий готовности:** Swagger открывается, справочник и партнёры в БД.

### Фаза 2 — Программные парсеры (3–5 ч)

**Приоритет: максимум данных за минимум времени**

| Файл | Парсер | Ожидаемый выход |
|---|---|---|
| Клиника 6.xlsx | openpyxl + pandas | код, название, цена |
| Клиника 7.xls | xlrd + pandas | название, 3 ценовых колонки |
| Клиника 8.xlsx | openpyxl, 2 листа | название, цены |
| Клиника 1.docx | python-docx | таблица ~2700 строк |
| Клиника 1.pdf | pdfplumber + regex | постраничный текст → строки |

Общие правила парсеров:
- Автоопределение строки заголовков (поиск «Наименование», «Код», «Стоимость»)
- Нормализация цен: убрать пробелы, «тг», «₸» → `decimal`
- Сохранение `service_name_raw`, `service_code_source`, `price_resident_kzt`, `price_nonresident_kzt`
- `raw_content` и `parse_log` в PriceDocument

**Критерий готовности:** ≥ 4 документа в статусе `done`, ≥ 3000 позиций в `price_items`.

### Фаза 3 — Matching Engine (2–3 ч)

Трёхэтапный конвейер (согласовано):

```
service_name_raw
      │
      ▼
[1] Exact Match     — normalize(lower, trim, ё→е, убрать пунктуацию)
      │ нет совпадения
      ▼
[2] Fuzzy Match     — RapidFuzz token_sort_ratio ≥ 85
      │ нет совпадения
      ▼
[3] LLM Match       — Gemini Flash: «сопоставь с одной из N услуг справочника»
      │ нет совпадения
      ▼
   unmatched queue
```

- Цель MVP: **≥ 70%** автосопоставления (требование ТЗ)
- Для LLM-match: батчами по 20–50 unmatched, top-10 кандидатов из fuzzy как контекст
- Поле `match_confidence` + `match_method` (exact / fuzzy / llm)

**Критерий готовности:** метрика `% matched` на дашборде, unmatched доступен через API.

### Фаза 4 — LLM-парсер для битых PDF (3–4 ч)

Клиники 2, 3, 4, 5 (+ дубли 2025/2026):

1. Конвертация страниц PDF → изображения (pdf2image) **или** отправка текста + скриншота в Gemini Flash (multimodal)
2. Промпт: извлечь JSON-массив `{name, code?, price_resident, price_nonresident?}`
3. Постобработка: валидация цен, дедупликация
4. Fallback: если LLM не справился — статус `needs_review`, фрагмент в лог

**Лимит API:** Flash + батчинг по страницам; кэшировать ответы в `raw_content` чтобы не гонять повторно.

**Критерий готовности:** все 10 файлов обработаны (хотя бы `done` или `needs_review`).

### Фаза 5 — API поиска (2–3 ч)

| Endpoint | MVP |
|---|---|
| `GET /search?q=` | Полнотекстовый поиск по услугам + цены партнёров |
| `GET /services/{id}/partners` | Клиники с ценами по услуге |
| `GET /partners/{id}/services` | Полный прайс клиники |
| `GET /unmatched` | Очередь несопоставленных |
| `POST /match` | Ручное сопоставление оператором |
| `POST /ingest` | Загрузка ZIP / одиночного файла |

OpenAPI/Swagger — обязательно (критерий оценки 15%).

### Фаза 6 — React UI (4–6 ч, параллельно с фазами 4–5)

**Языки: полный перевод RU + KZ** (i18n: `react-i18next`)

#### Публичная часть

| Экран | Функции |
|---|---|
| Главная | Поисковая строка с автодополнением (1200+ услуг) |
| Быстрые подсказки | Карточки: «Общий анализ крови (ОАК)», «Приём терапевта», «УЗИ» — клик → сразу результаты |
| Результаты поиска | Таблица: клиника, город, цена резидент, цена нерезидент, дата прайса |
| Карточка клиники | Контакты из `partners.json`, полный прайс, фильтр по категории |

#### Админка

| Экран | Функции |
|---|---|
| Дашборд | Документов обработано / % matched / unmatched / ошибки |
| Документы | Список файлов, статус, лог, кнопка «перепарсить» |
| Unmatched | Сырое название → предложенные совпадения → ручной match |
| Загрузка | Drag-and-drop ZIP |

### Фаза 7 — Деплой (2–3 ч)

- `Dockerfile` backend + frontend
- `docker-compose.yml` для локальной разработки

## 7. Риски и запасные планы

| Риск | Митигация |
|---|---|
| LLM не успевает / лимит API | Сначала programmatic; битые PDF — в `needs_review`, показать в админке |
| PDF клиники 1 слишком большой | Парсить DOCX 2024 как основу, PDF 2026 — выборочно или постранично |
| Низкий % match | Понизить fuzzy-порог до 80; LLM-батч для unmatched |
| React не успевает | API + Swagger как fallback-демо; UI наращивать |
| Railway падает | Локальный Docker как основной демо-стенд |
