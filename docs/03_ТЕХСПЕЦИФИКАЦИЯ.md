## 5. Пайплайн обработки документов

### 5.1 Router (`parsers/router.py`)

```python
def get_parser(file_path: str, file_format: str) -> BaseParser:
    BROKEN_PDF_CLINICS = {2, 3, 4, 5}

    if file_format in ("xlsx", "xls"):
        return XlsxParser() if file_format == "xlsx" else XlsParser()
    if file_format == "docx":
        return DocxParser()
    if file_format == "pdf":
        clinic_num = extract_clinic_number(file_path)
        if clinic_num in BROKEN_PDF_CLINICS:
            return LlmPdfParser()
        return PdfParser()  # pdfplumber
    raise UnsupportedFormatError(file_format)
```

### 5.2 Программные парсеры

#### Общий интерфейс (`parsers/base.py`)

```python
@dataclass
class ParsedRow:
    service_name_raw: str
    service_code_source: str | None
    price_resident_kzt: Decimal | None
    price_nonresident_kzt: Decimal | None
    currency_original: str = "KZT"

class BaseParser(ABC):
    @abstractmethod
    def parse(self, file_path: str) -> list[ParsedRow]: ...

    def find_header_row(self, rows: list[list]) -> int:
        """Ищем строку с ключевыми словами: наименование, код, стоимость, цена"""
        KEYWORDS = ["наименование", "название", "услуг", "стоимость", "цена", "код"]
        for i, row in enumerate(rows[:30]):
            text = " ".join(str(c).lower() for c in row if c)
            if sum(1 for kw in KEYWORDS if kw in text) >= 2:
                return i
        return 0
```

#### XLSX/XLS (`xlsx_parser.py`, `xls_parser.py`)

#### DOCX (`docx_parser.py`)

```python
def parse(self, file_path: str) -> list[ParsedRow]:
    doc = Document(file_path)
    table = doc.tables[0]
    rows = []
    for row in table.rows[1:]:  # пропуск заголовка
        cells = [c.text.strip() for c in row.cells]
        if len(cells) >= 3 and cells[0] and not cells[0].startswith("Раздел"):
            rows.append(ParsedRow(
                service_code_source=cells[0],
                service_name_raw=cells[1],
                price_resident_kzt=parse_price(cells[2])
            ))
    return rows
```

#### PDF структурированный (`pdf_parser.py`) — Клиника 1

```python
def parse(self, file_path: str) -> list[ParsedRow]:
    rows = []
    with pdfplumber.open(file_path) as pdf:
        for page in pdf.pages:
            text = page.extract_text() or ""
            for line in text.split("\n"):
                match = PRICE_LINE_RE.match(line)
                # Паттерн: "U1.1 Консультативный прием врача д.м.н. 16 600"
                if match:
                    rows.append(ParsedRow(...))
    return rows

PRICE_LINE_RE = re.compile(
    r'^([A-Z]?\d+[\.\d]*)\s+(.+?)\s+([\d\s]+)$'
)
```

### 5.3 LLM-парсер (`llm_pdf_parser.py`) — Клиники 2–5

#### Подход

1. Извлечь текст страницы через `pdfplumber` (даже битый — как контекст)
2. Конвертировать страницу в PNG через `pdf2image` (опционально, для multimodal)
3. Отправить в Gemini Flash
4. Получить JSON-массив позиций
5. Кэшировать ответ в `price_documents.raw_content`

#### Промпт (шаблон)

```
Ты — медицинский парсер прайс-листов. Извлеки все медицинские услуги и цены из текста.

Правила:
- Верни JSON-массив: [{"name": "...", "code": "...", "price_resident": 12600, "price_nonresident": 16380}]
- Цены — только числа в тенге, без пробелов и символов
- Если одна цена — price_nonresident = null
- Пропускай заголовки разделов без цен
- Восстанавливай испорченные слова по контексту (OAK → ОАК, yc.iv! → услуги)

Текст страницы:
---
{page_text}
---
```

## 6. Matching Engine

### 6.1 Нормализация строк (`matching/normalizer.py`)

```python
def normalize(text: str) -> str:
    text = text.lower().strip()
    text = text.replace("ё", "е")
    text = re.sub(r'[^\w\s]', ' ', text)   # убрать пунктуацию
    text = re.sub(r'\s+', ' ', text)      # схлопнуть пробелы
    # Убрать стоп-слова
    for sw in ["услуга", "исследование", "анализ", "прием", "приём", "консультация"]:
        text = text.replace(sw, "")
    return text.strip()
```

### 6.2 Exact Match

```python
def exact_match(raw: str, services: list[Service]) -> MatchResult | None:
    norm_raw = normalize(raw)
    for svc in services:
        if normalize(svc.service_name) == norm_raw:
            return MatchResult(svc.service_id, 1.0, "exact")
        for syn in svc.synonyms:
            if normalize(syn) == norm_raw:
                return MatchResult(svc.service_id, 1.0, "exact")
    return None
```

### 6.3 Fuzzy Match (RapidFuzz)

```python
from rapidfuzz import fuzz, process

FUZZY_THRESHOLD = 85  # конфигурируемый

def fuzzy_match(raw: str, services: list[Service]) -> MatchResult | None:
    choices = {s.service_id: normalize(s.service_name) for s in services}
    result = process.extractOne(
        normalize(raw), choices.values(),
        scorer=fuzz.token_sort_ratio
    )
    if result and result[1] >= FUZZY_THRESHOLD:
        service_id = [k for k, v in choices.items() if v == result[0]][0]
        return MatchResult(service_id, result[1] / 100, "fuzzy")
    return None
```

### 6.4 LLM Match

Применяется:
- На этапе **постобработки unmatched** (батчами)
- Для аббревиатур: ОАК, УЗИ, ЭКГ, МРТ, КТ, ПЦР

```python
MATCH_PROMPT = """
Сопоставь название услуги из прайса клиники с одной услугой из справочника.

Название из прайса: "{raw_name}"

Кандидаты из справочника:
{candidates}

Верни JSON: {{"service_id": "uuid или null", "confidence": 0.0-1.0, "reason": "..."}}
Если ни один кандидат не подходит — service_id: null.
"""
```

Кандидаты для промпта — top-10 из fuzzy с порогом ≥ 60.

### 6.5 Полный pipeline (`matching/engine.py`)

```python
def match_item(raw_name: str, services: list[Service]) -> MatchResult:
    result = exact_match(raw_name, services)
    if result: return result

    result = fuzzy_match(raw_name, services)
    if result and result.confidence >= 0.85: return result

    result = llm_match(raw_name, services, fuzzy_candidates=...)
    if result and result.confidence >= 0.80: return result

    return MatchResult(None, 0.0, "unmatched")
```

## 7. Валидация (`validators/price_validator.py`)

| Проверка | Условие | Действие |
|---|---|---|
| `price_positive` | price > 0 | Иначе: skip + log |
| `price_is_number` | isinstance decimal | Иначе: needs_review |
| `nonresident_gte_resident` | nonres >= res | Иначе: warning flag |
| `name_not_empty` | len(name) > 2 | Иначе: skip |
| `date_not_future` | effective_date <= today | Иначе: warning |
| `price_anomaly` | change > 50% vs prev | Flag + needs_review |
| `duplicate` | same partner+service+date | Archive old, keep new |
| `no_data` | 0 rows parsed | status = error |

```yaml
    ports: ["5432:5432"]
    volumes: [pgdata:/var/lib/postgresql/data]

  api:
    build: ./backend
    ports: ["8000:8000"]
    environment:
      DATABASE_URL: postgresql://medpartners:${DB_PASSWORD:-devpassword}@db:5432/medpartners
      GEMINI_API_KEY: ${GEMINI_API_KEY}
    depends_on: [db]
    volumes: [./data:/app/data]

  frontend:
    build: ./frontend
    ports: ["3000:3000"]
    environment:
      VITE_API_URL: http://localhost:8000

volumes:
  pgdata:
```
